# 병목 문장 독해 실험

한국어 과학 지문의 **문법적 병목 강도**가 읽기 시간과 이해도에 미치는 영향을 측정하는 웹 실험 도구.

**실험 페이지:** https://shingidong.github.io/reading_language/

---

## 핵심 설계

같은 내용(명제)을 **문법 구조만 바꿔** 4가지 병목 수준으로 만들고,
참가자마다 **명제별로 수준을 따로 무작위 배정**한다.

```
             명제1      명제2      명제3      명제4      명제5
참가자 A      B3        B1        B4        B2        B4
참가자 B      B1        B4        B2        B3        B1
참가자 C      B4        B2        B3        B1        B2   ...
```

→ **한 사람이 여러 병목 수준을 다 읽는다** (개인 내 설계).
읽기 속도·국어 실력 같은 개인차가 자동으로 상쇄되므로, **30명 정도만 모여도 결론이 나온다.**
(참가자마다 B1~B4가 최소 1번씩은 반드시 나오도록 보장한다.)

### 변인

- **독립변인**: 문장의 **병목 점수** (1.0 ~ 10.0, 연속 변인)
- **종속변인**:
  - **음절당 읽기 시간 (ms)** ← 글자 수 차이를 보정한 값. **비교는 반드시 이 값으로!**
  - 재읽기 횟수
  - 해당 명제를 근거로 하는 문항의 정답 여부
  - 확신도 (확신함 / 잘 모름 / 찍었음) → '이해 착각'(오답인데 확신) 탐지

## 병목 점수는 어떻게 매기는가

각 문장의 문법 자질을 세어서 가중합으로 계산한다. ([data.js](data.js) 참고)

```
병목 점수(원점수) = 2.0 × 안긴절 수
                 + 1.5 × 명사화 수
                 + 1.2 × 피동 표현 수
                 + 0.8 × 절 밀도(절 수 ÷ 문장 수)
```

이후 20개 버전(명제 5 × 수준 4) 전체를 **1.0~10.0으로 정규화**한다.

| 자질 | 왜 병목인가 | 예 |
|---|---|---|
| 안긴절 | 미완결 구조를 작업기억에 쌓아둬야 함 | "…에 의해 결정되는 …라 불리는 값" |
| 명사화 | 용언을 명사로 압축해 사건 구조를 숨김 | 부피가 변함 → **부피 변화** |
| 피동 | 행위자를 숨겨 누가 무엇을 했는지 흐림 | 조절**된다**, 서술**된다** |
| 절 밀도 | 한 문장에 절을 몇 개나 욱여넣었는가 | |

### 실제 점수표 (검증 완료: 모든 명제에서 B1 < B2 < B3 < B4)

| 명제 | B1 | B2 | B3 | B4 |
|---|---|---|---|---|
| 1. 열팽창의 정의 | 1.0 | 2.4 | 3.3 | 4.1 |
| 2. 열팽창 계수 | 1.0 | 2.4 | 5.8 | 7.3 |
| 3. 바이메탈이 휘는 원리 | 3.1 | 4.6 | 8.3 | 9.4 |
| 4. 최대 이동 거리 | 3.4 | 3.9 | 9.8 | 10.0 |
| 5. 온도 조절 장치 | 1.1 | 1.5 | 4.4 | 7.4 |

B1~B4는 **설계 의도를 나타내는 라벨일 뿐**이고, 실제 분석에 쓰는 변인은 이 **연속 점수**다.
덕분에 "B4가 B1보다 느리다"가 아니라 **"병목 점수가 1점 오를 때 음절당 읽기 시간이 몇 ms 늘어난다"**는
회귀 결과를 낼 수 있다.

## 문항 ↔ 명제 매핑

각 문항은 **어느 명제를 읽어야 풀리는지**가 지정되어 있다. 그래서 정답 여부를
그 명제의 병목 점수에 직접 연결할 수 있다.

| 문항 | 유형 | 근거 명제 |
|---|---|---|
| Q1 | 회상(표층) | 1 |
| Q2 | 회상(표층) | 2 |
| Q3 | 통합(텍스트 기저) | 3 |
| Q4 | 통합(텍스트 기저) | 4 |
| Q5 | 추론(상황 모형) | 4 |
| Q6 | 추론(상황 모형) | 5 |

(Kintsch의 3층위 이해 모형에 대응)

## 데이터 수집 설정 (필수)

`config.js`의 `SHEETS_WEBHOOK_URL`이 **비어 있으면 응답이 어디에도 저장되지 않습니다.**

1. 구글 드라이브에서 **새 스프레드시트 생성**
2. **확장 프로그램 → Apps Script**
3. [google_apps_script.js](google_apps_script.js) 코드 전체를 복사 붙여넣기 (기존 코드 삭제)
4. **저장** → **배포 → 새 배포**
   - 유형: **웹 앱**
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자** ← 반드시!
5. 발급된 URL을 `config.js`에 붙여넣기

```js
const SHEETS_WEBHOOK_URL = "https://script.google.com/macros/s/여기/exec";
```

```bash
git add config.js && git commit -m "웹훅 설정" && git push
```

> 정적 사이트라 웹훅 URL이 소스에 노출됩니다. 이 URL은 시트에 **쓰기만** 가능하고 읽지는 못하므로
> 학급 단위 탐구에는 문제없습니다.

## 데이터 분석

구글 시트에 4개 시트가 자동 생성됩니다.

### ★ `analysis` — 이 시트 하나로 분석이 끝납니다

**한 행 = 참가자 한 명이 명제 하나를 읽은 기록.** 30명이면 150행이 쌓입니다.

| 참가자ID | 명제ID | 병목수준 | **병목점수** | 음절수 | **음절당읽기시간(ms)** | 재읽기 | **해당정답률(%)** |
|---|---|---|---|---|---|---|---|
| 3-5-KDS | 1 | B3 | 3.3 | 37 | 33.4 | 0 | 100 |
| 3-5-KDS | 4 | B4 | 10.0 | 57 | 61.2 | 1 | 50 |
| … | | | | | | | |

**바로 할 수 있는 것:**

1. **산점도** — x축 `병목점수`, y축 `음절당읽기시간(ms)` → 우상향하면 가설 지지
2. **추세선/회귀** — 기울기가 "병목 1점당 늘어나는 ms"
3. **산점도 2** — x축 `병목점수`, y축 `해당정답률(%)` → 우하향하면 가설 지지

```
# 병목 수준별 평균 음절당 읽기 시간
=AVERAGEIF(analysis!G:G, "B1", analysis!P:P)
=AVERAGEIF(analysis!G:G, "B4", analysis!P:P)

# 병목 점수와 읽기 시간의 상관계수
=CORREL(analysis!H:H, analysis!P:P)

# 병목 점수와 정답률의 상관계수
=CORREL(analysis!H:H, analysis!T:T)

# 회귀 기울기 (병목 1점당 늘어나는 음절당 ms)
=SLOPE(analysis!P:P, analysis!H:H)
```

### 나머지 시트

- **quiz_responses** — 문항별 응답. 각 문항에 근거 명제의 병목 점수가 붙어 있음
- **participants** — 참가자별 요약. B1~B4 각각의 음절당 읽기시간·정답률 (개인 내 비교)
- **raw_json** — 원본 백업 (이동 이벤트 로그 포함)

## 결과 해석 가이드

| 관찰 | 의미 |
|---|---|
| 병목점수 ↑ → 음절당 읽기시간 ↑ | 문법 구조가 처리 부담을 늘린다 (작업기억 가설 지지) |
| 병목점수 ↑ → 정답률 ↓ | 처리 부담이 **이해 실패**로 이어진다 |
| 병목점수 ↑ → 읽기시간 ↑ **인데 정답률은 유지** | 더 오래 읽어서 **보상**했다는 뜻 (시간 비용만 발생) |
| 병목점수 ↑ → '이해 착각'(오답+확신) ↑ | 병목 문장은 **틀렸는데도 맞았다고 믿게** 만든다 ← 가장 흥미로운 결과 |

세 번째와 네 번째가 나오면 "어려운 글은 그냥 어렵다"보다 훨씬 할 말이 많은 탐구가 됩니다.

## 알려진 한계 (보고서에 쓰면 좋음)

- **명제 5개뿐** — 병목 점수의 표본이 적어 회귀선이 소수의 문장에 좌우됨
- **명제 4의 B3(9.8)와 B4(10.0)가 거의 붙어 있음** — 이 둘의 차이는 검출하기 어려움
- **명제 단위 제시** — 문장이 아니라 명제 덩어리 단위로 시간을 재므로, 문장 내부의 어느 지점에서 병목이 걸리는지는 알 수 없음
- 병목 점수의 **가중치(2.0 / 1.5 / 1.2 / 0.8)는 이론에 근거한 임의값**이지 실증적으로 추정한 값이 아님
